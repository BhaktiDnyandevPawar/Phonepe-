# PhonePe Transaction Analysis Dashboard

##  Project Overview

This project focuses on analyzing PhonePe transaction data to understand digital payment trends across India. It includes data cleaning, visualization, SQL integration, and an interactive dashboard built using Streamlit.

##  Business Objective

* Identify top-performing states, districts, and transaction categories
* Analyze transaction growth over years and quarters
* Understand user behavior and engagement patterns
* Support data-driven decision-making for business expansion

##  Dataset

* Source: PhonePe Pulse GitHub Repository
* Data includes:

  * State-wise transactions
  * Year & Quarter data
  * Transaction type
  * Transaction count & amount

##  Technologies Used

* Python
* Pandas
* NumPy
* Matplotlib
* Seaborn
* SQL (SQLite)
* Streamlit

##  Project Workflow

###  Data Collection

Extracted data from PhonePe Pulse GitHub repository

###  Data Cleaning

* Checked missing values
* Removed duplicates
* Structured dataset for analysis

### Exploratory Data Analysis (EDA)

* Analyzed trends using 15+ visualizations
* Compared states, years, and transaction types

###  SQL Integration

* Stored cleaned data into SQLite database
* Performed queries for insights

###  Dashboard Development

* Built interactive dashboard using Streamlit
* Added filters (State, Year)
* Included charts, KPIs, and map visualization

##  Key Insights

* Maharashtra, Karnataka, and Telangana are top-performing states
* Peer-to-peer and merchant payments dominate transactions
* Digital payments show strong growth year by year
* Some states have lower adoption, indicating growth opportunities
* Seasonal variations observed across quarters

##  Features

✔ Clean and structured dataset
✔ 15+ data visualizations
✔ SQL database integration
✔ Interactive Streamlit dashboard
✔ Filters for dynamic analysis
✔ Map visualization of transactions

